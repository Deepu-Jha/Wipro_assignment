# Wipro Online Training – Automation Testing Journey

Welcome to my Wipro Online Training repository.

This repository contains the projects, assignments, and frameworks that I developed while learning and practicing Automation Testing concepts during my training. The goal of these projects was to gain hands-on experience with industry-standard tools and frameworks used by SDETs and QA Automation Engineers.

## What You Will Find in This Repository

### GUIAutomationFramework

A complete Selenium automation framework developed using Java, TestNG, Maven, and the Page Object Model (POM) design pattern. This project includes reusable components, test execution, and reporting features.

### Rest_Api_Project

API automation testing project built using REST Assured. It covers API requests, response validation, status code verification, and JSON data handling.

### Jenkins_wipro

Practice project focused on Jenkins CI/CD integration. It demonstrates how automated test execution can be triggered through Jenkins pipelines and jobs.

### Selenium & Selenium_1

A collection of Selenium WebDriver programs covering browser automation, web element handling, synchronization, and automation concepts.

### cucumber_day30

BDD automation project using Cucumber. It includes Feature Files, Step Definitions, and Gherkin scenarios for behavior-driven testing.

### Docker_day_34

Practice project for understanding Docker basics and Selenium Grid execution using Docker containers.

### EccomerceFramework

An end-to-end automation framework created for an e-commerce application using Selenium, Java, TestNG, and Maven.

### php

Practice exercises and learning activities completed during the training period.

### wipro_2026

Additional assignments, coding exercises, and learning materials completed as part of the Wipro training program.

## How to Run the Projects

1. Clone the repository:

```bash
git clone https://github.com/Deepu-Jha/Wipro_assignment.git
```

2. Import the required project into Eclipse.

3. Update Maven dependencies:

```bash
mvn clean install
```

4. Execute the test cases:

```bash
mvn test
```
